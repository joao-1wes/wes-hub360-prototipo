
Como usuário do HUB,

eu quero organizar agentes por organização e por contexto de projeto,

para que os agentes de diferentes projetos e seus respectivos contextos, não compartilhem os mesmos tópicos, informações ou referências entre si de forma indevida.


Organização por contexto

O HUB deve permitir organizar agentes dentro de uma pasta que terá o título do projeto que o usuário escolheu, agentes com contextos específicos dentro desses projetos em que estão relacionados vinculados a uma organização.

O usuário deve conseguir criar contextos distintos para representar projetos, frentes ou áreas de trabalho.

Um contexto deve funcionar como um espaço próprio para agrupar os agentes relacionados àquele escopo.

A criação de novos agentes deve acontecer sempre associada a um contexto previamente definido.


Isolamento entre contextos
Agentes criados em um contexto não devem compartilhar referências com agentes de outros contextos.

Informações associadas a um contexto devem permanecer restritas àquele espaço.

O uso de um agente não deve considerar conteúdos, históricos ou materiais de outros contextos de forma indevida.

O isolamento entre contextos deve acontecer mesmo quando eles estiverem dentro da mesma organização.


Separação entre organizações
Agentes de uma organização não devem ser exibidos em outra organização.

Os contextos criados em uma organização devem ser independentes dos contextos de outras organizações.

A estrutura do HUB deve impedir confusão entre agentes pertencentes a organizações diferentes.


Experiência de organização
A navegação deve deixar claro em qual organização, projeto e contexto o usuário está atuando.

O usuário deve conseguir identificar facilmente a qual contexto cada agente pertence.

A visualização dos agentes deve refletir essa separação de forma simples e intuitiva.

A experiência de organização deve ser semelhante à lógica de espaços independentes.
